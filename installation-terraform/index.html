Installation

Prerequisites

Check you've done all the preparation and that you can connect to a running cluster. You should have all the required CLI tools and a running local cluster (using either k3d, kind, docker desktop, rancher desktop or minikube).

To check:

kubectl cluster-info --context <your cluster name>

Note

When you created your local cluster on windows and now running WSL, copy your windows context to your WSL installation.

cp /mnt/c/users/<your username>/.kube/config ~/.kube/config

Update Local DNS

Since we are running completely locally, we want to add some entries to the hosts file to forward domainnames to our locally running cluster. On Windows, open C:\Windows\system32\drivers\etc\hosts in an editor as Administrator and add the following entries. On Linux edit your /etc/hosts file as root.

127.0.0.1 k8s.local infrastructure.k8s.local traefik.infrastructure.k8s.local traefik.k8s.local

Generate a root certificate

Assuming the root certificate is already created with mkcert -install, then

cd src/terraform
./generate-certificate.ps1

Next run the installer in a Linux/WSL shell:

cd src/terraform
./generate-certificate.sh

Installer

Edit the terraform.tfvars as you see fit.

cd src/terraform
helm repo update
terraform init
terraform apply -auto-approve

Next run the installer in a Linux/WSL shell:

cd src/terraform
helm repo update
terraform init
terraform apply -auto-approve

The traefik dashboard should now be available at https://traefik.infrastructure.k8s.local
